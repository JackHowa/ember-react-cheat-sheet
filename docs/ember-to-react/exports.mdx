# Exports

In React when exporting a component, we must use default exports vs named exports.

## Ember

In ember we already use default exports:
```js title="component.js" showLineNumbers
export default class OrganizationSelector extends Component {
  @service('user') userService;
  matcher = (item, searchString) =>
    defaultMatcher(`${item.shortName} - ${item.name}`, searchString);

  @action
  isGeoRestricted(organization) {
    const orgRestrictions =
      get(organization?.organizationStatus || {}, 'restrictions') || [];
    return (
      this.userService?.hasExpelEmail &&
      orgRestrictions.some((r) => r.includes('GEO'))
    );
  }
}
```

```hbs title="template.hbs"  showLineNumbers
 <OrganizationSelector
        class="organization-selector ml-3"
        @organizations={{this.organizations}}
        @selected={{or
          this.organizationService.current
          (find-by "id" undefined this.organizations)
        }}
        @organizationSelected={{@setCurrentOrganization}}
      />
```

## React

This means doing something like this when importing/exporting:

Import:
```jsx 
import organizationSelector from './organizationSelector.jsx';
```

Export:
```jsx 
export default organizationSelector;
```

**Do not do the below**:

Import:
```jsx 
import { organizationSelector } from './organizationSelector.jsx';
```

Export:
```jsx 
export { organizationSelector };
```
